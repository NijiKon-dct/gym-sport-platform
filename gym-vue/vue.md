# 健身场馆预约系统 - 前端系统设计方案

## 1. 项目概述

健身场馆预约系统是一个基于Vue 3 + TypeScript + Pinia的现代化Web应用，旨在为用户提供便捷的健身场馆预约服务。系统采用前后端分离架构，前端负责用户界面展示和交互，后端提供API接口支持。

### 1.1 技术栈

- **框架**: Vue 3.5.25 (Composition API)
- **语言**: TypeScript 5.9.0
- **构建工具**: Vite 7.2.4
- **状态管理**: Pinia 3.0.4
- **路由**: Vue Router 4.6.3
- **UI组件库**: Element Plus 2.11.8
- **HTTP客户端**: Axios 1.13.2
- **代码规范**: ESLint + Prettier

## 2. 系统架构设计

### 2.1 整体架构

系统采用MVVM架构模式，结合Vue 3的Composition API实现组件化开发。

```
┌─────────────────────────────────────────────────────┐
│                    前端应用                           │
├─────────────────────────────────────────────────────┤
│  视图层(Views)                                       │
│  ├── HomeView.vue      (首页)                        │
│  ├── LoginView.vue     (登录页)                      │
│  ├── RegisterView.vue  (注册页)                      │
│  └── AboutView.vue     (关于页)                      │
├─────────────────────────────────────────────────────┤
│  组件层(Components)                                  │
│  ├── 通用组件(BaseButton, BaseInput等)               │
│  └── 业务组件(场馆卡片、预约表单等)                   │
├─────────────────────────────────────────────────────┤
│  状态管理层(Stores)                                   │
│  ├── user.ts           (用户状态)                    │
│  └── venue.ts          (场馆状态)                    │
├─────────────────────────────────────────────────────┤
│  服务层(Services/API)                                 │
│  ├── user.ts           (用户API)                     │
│  └── venue.ts          (场馆API)                     │
├─────────────────────────────────────────────────────┤
│  工具层(Utils)                                        │
│  ├── common.ts         (通用工具)                    │
│  ├── request.ts        (HTTP请求封装)                │
│  └── validation.ts     (表单验证)                    │
└─────────────────────────────────────────────────────┘
```

### 2.2 数据流设计

系统采用单向数据流设计，状态管理使用Pinia实现：

```
用户操作 → 组件事件 → Store Actions → API请求 → 后端响应
                ↓
            状态更新 → 组件重新渲染 → 视图更新
```

## 3. 核心功能模块

### 3.1 用户认证模块

**功能描述**: 实现用户登录、注册、登出等功能。

**技术实现**:
- 使用Pinia管理用户状态
- 本地存储token和用户信息
- 路由守卫实现访问控制

### 3.2 场馆展示模块

**功能描述**: 展示场馆列表、详情、筛选和搜索功能。

**技术实现**:
- 响应式布局适配不同设备
- 虚拟滚动优化大量数据展示
- 防抖搜索减少请求频率

### 3.3 场馆预约模块

**功能描述**: 实现场馆预约、取消预约、查看预约记录等功能。

**技术实现**:
- 表单验证确保数据完整性
- 时间选择器避免冲突预约
- 状态管理实时更新预约信息

### 3.4 场馆信息管理模块

- 场馆基本信息维护（名称、地址、开放时间等）
- 场地类型管理（篮球场、羽毛球场、健身房等）
- 场地详情管理（场地编号、设施情况、容纳人数等）
- 场馆图片与描述管理

### 3.5 预约管理模块

- 场地实时状态查询
- 预约申请与创建
- 预约记录查询与管理
- 预约取消与修改
- 冲突检测与处理
- 预约时间规则设置

### 3.6 社交互动模块

- 运动动态发布（文字、图片、视频）
- 动态点赞、评论、分享
- 用户关注功能
- 运动圈子/群组管理
- 运动伙伴匹配推荐

### 3.7 消息通知模块

- 预约状态变更通知
- 系统消息推送
- 社交互动通知（评论、点赞、关注）
- 消息中心与历史记录

## 5. 状态管理设计

### 5.1 用户状态管理

**状态变量**:
- userInfo: 用户基本信息
- token: 认证令牌
- isLoggedIn: 登录状态

**操作方法**:
- userLogin: 用户登录
- userRegister: 用户注册
- logout: 用户登出
- updateUserInfo: 更新用户信息

### 5.2 场馆状态管理

**状态变量**:
- venues: 场馆列表
- currentVenue: 当前选中的场馆
- bookings: 用户预约记录
- loading: 加载状态

**计算属性**:
- filteredVenues: 筛选后的场馆列表
- availableVenues: 可预约的场馆

**操作方法**:
- fetchVenues: 获取场馆列表
- fetchVenueById: 根据ID获取场馆详情
- makeBooking: 创建预约
- cancelBooking: 取消预约
