# 体育馆预约管理系统 - 代码讲解

## 一、项目架构概述

本项目采用**前后端分离**架构，前端基于 Vue 3 + TypeScript + Vite，后端基于 Spring Boot 3 + JPA + MySQL。

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 + TypeScript | 渐进式 JavaScript 框架，类型安全 |
| 构建工具 | Vite | 下一代前端构建工具 |
| 状态管理 | Pinia | Vue 官方状态管理库 |
| UI组件库 | Element Plus + TDesign | 企业级 UI 组件 |
| 路由 | Vue Router | Vue 官方路由管理器 |
| 通信 | Axios + WebSocket(STOMP) | HTTP 请求 + 实时消息推送 |
| 后端框架 | Spring Boot 3.5.8 | Java 企业级开发框架 |
| ORM | Spring Data JPA | 数据访问层 |
| 数据库 | MySQL | 关系型数据库 |
| 安全 | Spring Security | 安全框架（密码加密） |

## 二、前端核心代码讲解

### 2.1 路由配置 (`src/router/index.ts`)

**核心设计要点：**

```typescript
// 路由守卫 - 登录状态控制
router.beforeEach((to, from, next) => {
const userStore = useUserStore()
const isLoggedIn = userStore.isLoggedIn
const isAdmin = userStore.userInfo?.isAdmin

// 已登录用户禁止访问登录/注册页
if (isLoggedIn && (to.name === 'login' || to.name === 'register')) {
    if (isAdmin) {
    return next({ name: 'admin-statistics' })
    }
    return next({ name: 'home' })
}

// 未登录用户访问需要登录的页面时重定向
if (!isLoggedIn && !to.meta.allowGuest) {
    return next({ name: 'login', query: { redirect: to.fullPath } })
}

// 管理员权限校验
if (to.meta.requiresAdmin && !isAdmin) {
    next({ name: 'home' })
} else {
    next()
}
})
```

**设计思路：**
- 使用 `meta` 字段标记路由属性（`allowGuest`、`requiresAdmin`）
- 通过 Pinia 的 `useUserStore` 获取用户登录状态
- 实现完整的权限控制逻辑：登录拦截、管理员权限校验、登录后重定向

### 2.2 聊天状态管理 (`src/stores/chat.ts`)

这是本项目**最核心的实时通信模块**，实现了完整的私聊功能。

#### 2.2.1 WebSocket 连接机制

```typescript
const connectWebSocket = () => {

    })

    client.subscribe(`/topic/user.${userId}.typing`, (frame) => { /* ... */ })
    
    // 订阅已读回执
    client.subscribe(`/topic/user.${userId}.read-receipts`, (frame) => { /* ... */ })

```

**关键设计点：**
- 使用 **SockJS** 作为 WebSocket 降级方案，兼容不支持 WebSocket 的浏览器
- 通过 URL 参数传递 `userId`，后端据此识别用户身份
- 采用 **topic 模式**实现消息推送，每个用户订阅专属频道 `/topic/user.{userId}.messages`
- 实现断线自动重连机制（`reconnectDelay: 5000`）

#### 2.2.2 消息去重机制

```typescript
// 消息去重逻辑 - 通过 ID 或内容+发送者+时间戳判断
const existing = chatMessages.find(msg => 
msg.id === body.id || 
(msg.content === body.content && 
    msg.senderId === body.senderId && 
    Math.abs(new Date(msg.createdAt).getTime() - new Date(body.createdAt).getTime()) < 1000)
)

if (existing) {
// 更新已有消息的状态
existing.id = body.id
existing.localStatus = body.localStatus
existing.isRead = body.isRead
} else {
// 添加新消息
chatMessages.push(body)
}
```

**设计原因：**
- 消息可能通过 HTTP 响应和 WebSocket 推送两种方式到达前端
- 避免重复显示同一条消息
- 保证消息状态（已读/送达）的正确更新

#### 2.2.3 本地消息状态管理

```typescript
interface ChatMessage {
id: number
chatId: number
senderId: number
receiverId: number
content: string
type: ChatMessageType
createdAt: string
isRead: boolean
// 本地状态：sent(已发出)、delivered(已送达)、read(已读)
localStatus?: 'sent' | 'delivered' | 'read'
}
```

**状态流转：**
1. `sent`：消息已发送到服务器
2. `delivered`：服务器已推送消息给对方（收到 WebSocket 回推）
3. `read`：对方已读（收到已读回执）

### 3.1 WebSocket 配置 (`config/WebSocketConfig.java`)

```java
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

@Override
public void configureMessageBroker(MessageBrokerRegistry config) {
    // 启用简单消息代理，支持 /queue 和 /topic 前缀
    config.enableSimpleBroker("/queue", "/topic");
    // 客户端发送消息的前缀
    config.setApplicationDestinationPrefixes("/app");
    // 用户目的地前缀
    config.setUserDestinationPrefix("/user");
}

@Override
public void registerStompEndpoints(StompEndpointRegistry registry) {
    registry.addEndpoint("/ws-chat")
            .setHandshakeHandler(new DefaultHandshakeHandler() {
                @Override
                protected Principal determineUser(ServerHttpRequest request,
                                                WebSocketHandler wsHandler,
                                                Map<String, Object> attributes) {
                    // 从请求参数中提取 userId 作为 Principal
                    if (request instanceof ServletServerHttpRequest servletRequest) {
                        String userId = servletRequest.getServletRequest().getParameter("userId");
                        if (userId != null && !userId.isBlank()) {
                            return () -> userId;
                        }
                    }
                    return super.determineUser(request, wsHandler, attributes);
                }
            })
            .setAllowedOriginPatterns("*")
            .withSockJS();  // 启用 SockJS 支持
}
}
```

**关键技术点：**
- 使用 **STOMP 协议**（Simple Text Oriented Messaging Protocol）实现消息路由
- 通过自定义 `HandshakeHandler` 从 URL 参数获取 `userId`，作为 STOMP 会话的用户标识
- 配置 `/topic` 前缀用于广播消息

### 3.2 聊天服务 (`service/ChatService.java`)

#### 3.2.1 消息发送与推送

```java
public ChatMessageDto sendMessage(SendMessageRequest request) {
ChatSession session = chatSessionRepository.findById(request.chatId())
        .orElseThrow(() -> new ResourceNotFoundException("聊天不存在"));
User sender = userService.findById(request.senderId());
User receiver = userService.findById(request.receiverId());

// 验证双方是否为聊天参与者
ensureParticipant(session, sender.getId());
ensureParticipant(session, receiver.getId());

// 创建并保存消息
Message message = Message.builder()
        .chat(session)
        .sender(sender)
        .receiver(receiver)
        .content(request.content())
        .type(msgType)
        .build();
Message saved = messageRepository.save(message);

// 更新聊天会话的最后消息
session.setLastMessage(lastMessagePreview);
session.setLastMessageTime(saved.getCreatedAt());
// 更新未读计数
if (session.getUserA().getId().equals(receiver.getId())) {
    session.setUnreadCountForUserA(session.getUnreadCountForUserA() + 1);
} else {
    session.setUnreadCountForUserB(session.getUnreadCountForUserB() + 1);
}

// 通过 WebSocket 推送消息
ChatMessageDto dto = toMessageDto(saved);
try {
    // 推送给接收方
    messagingTemplate.convertAndSend(
            "/topic/user." + dto.receiverId() + ".messages",
            dto
    );
    // 推送给发送方（状态同步）
    messagingTemplate.convertAndSend(
            "/topic/user." + dto.senderId() + ".messages",
            dto
    );
} catch (Exception e) {
    // 推送失败不影响主流程
}

return dto;
}
```

**设计亮点：**
- 使用 `SimpMessagingTemplate` 实现消息推送
- 同时推送给发送方和接收方，保证两端状态一致
- 异常捕获确保推送失败不影响消息持久化

#### 3.2.2 已读回执机制

```java
public ChatDto markAsRead(MarkReadRequest request) {
ChatSession session = chatSessionRepository.findById(request.chatId())
        .orElseThrow(() -> new ResourceNotFoundException("聊天不存在"));

// 将该用户收到的所有消息标记为已读
messageRepository.findByChatIdOrderByCreatedAtAsc(request.chatId()).stream()
        .filter(message -> message.getReceiver().getId().equals(request.userId()))
        .forEach(message -> {
            message.setIsRead(true);
            message.setReadAt(LocalDateTime.now());
        });

// 清除未读计数
if (session.getUserA().getId().equals(request.userId())) {
    session.setUnreadCountForUserA(0);
} else {
    session.setUnreadCountForUserB(0);
}

// 向对方推送已读回执
Long otherUserId = session.getUserA().getId().equals(request.userId())
        ? session.getUserB().getId()
        : session.getUserA().getId();
try {
    messagingTemplate.convertAndSend(
            "/topic/user." + otherUserId + ".read-receipts",
            new ReadReceiptDto(request.chatId(), request.userId(), LocalDateTime.now())
    );
} catch (Exception e) { /* 忽略推送异常 */ }

return toChatDto(session, request.userId());
}
```

### 3.3 预约服务 (`service/BookingService.java`)

#### 3.3.1 预约冲突检测

```java
public BookingResponse create(BookingRequest request) {
User user = userService.findById(request.userId());
Venue venue = venueRepository.findById(request.venueId())
        .orElseThrow(() -> new ResourceNotFoundException("场馆不存在"));

// 验证时间范围有效性
validateTimeRange(request.startTime(), request.endTime());
// 检测时间段是否与已有预约重叠
ensureNotOverlapping(request.venueId(), request.date(), request.startTime(), request.endTime());

// 创建预约记录
Booking booking = Booking.builder()
        .user(user)
        .venue(venue)
        .date(request.date())
        .startTime(request.startTime())
        .endTime(request.endTime())
        .status(BookingStatus.PENDING)
        .price(price)
        .paid(false)
        .remark(request.remark())
        .build();

return toResponse(bookingRepository.save(booking));
}

private void ensureNotOverlapping(Long venueId, LocalDate date, LocalTime start, LocalTime end) {
long count = bookingRepository.countOverlappingBookings(venueId, date, start, end);
if (count > 0) {
    throw new BadRequestException("您选择的时间段与已有预约重叠");
}
}
```

#### 3.3.2 定时任务 - 自动取消超时未付款预约

```java
@Scheduled(cron = "${booking.auto-cancel.cron:0 */1 * * * *}")
public void autoCancelExpiredUnpaidBookings() {
LocalDateTime now = LocalDateTime.now();
// 仅处理「待确认且未付款」的预约
List<Booking> unpaidBookings = bookingRepository.findByPaidFalseAndStatus(BookingStatus.PENDING);

for (Booking booking : unpaidBookings) {
    LocalDateTime bookingEnd = LocalDateTime.of(booking.getDate(), booking.getEndTime());
    if (bookingEnd.isBefore(now)) {
    booking.setStatus(BookingStatus.CANCELLED);
    booking.setRemark("系统自动取消：预约时间已结束且未完成付款");
    }
}
}
```

**设计要点：**
- 使用 `@Scheduled` 注解实现定时任务
- cron 表达式可通过配置文件动态调整（默认每分钟执行）
- 仅处理 `PENDING` 状态且未付款的预约，避免误取消用户主动取消的订单

public AuthResponse login(LoginRequest request) {
User user = userRepository.findByAccount(request.account())
        .orElseThrow(() -> new ResourceNotFoundException("账号不存在"));

// 使用 BCrypt 验证密码

public AuthResponse register(RegisterRequest request) {

    throw new BadRequestException("账号已存在");
}

        .account(request.account())
        .username(request.username())
        .password(passwordEncoder.encode(request.password()))  // BCrypt 加密
        .email(request.email())
        .phone(request.phone())
        .isStudent(request.isStudent())
        .school(request.school())
        .nickname(request.username())
        .avatar(/* 默认头像 */)
        .isAdmin(false)
        .build();
user.getRoles().add("ROLE_USER");

User saved = userRepository.save(user);
String token = "mock-token-" + UUID.randomUUID();
return new AuthResponse(token, toDto(saved));
}

### 3.5 安全配置 (`config/SecurityConfig.java`)

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http.csrf(csrf -> csrf.disable())  // 禁用 CSRF（前后端分离架构）
            .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())  // 允许所有请求
            .httpBasic(Customizer.withDefaults());
    return http.build();
}

@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();  // BCrypt 密码加密器
}
}
```

**设计说明：**
- 禁用 CSRF 是因为前后端分离架构下，前端通过 Token 认证而非 Session
- 当前配置允许所有请求访问（实际生产环境应根据需要配置权限控制）
- 使用 BCrypt 算法进行密码加密，安全性高

## 五、核心业务流程

### 5.1 预约流程

```
用户选择场馆 → 选择日期时间 → 系统检测冲突 → 创建预约 → 付款 → 确认预约
            ↓
        冲突检测失败 → 返回错误提示
```

### 5.2 实时聊天流程

```
用户A发送消息
    ↓
后端保存消息到数据库
    ↓
更新聊天会话最后消息和未读计数
    ↓
WebSocket 推送给用户A和用户B
    ↓
前端收到消息更新界面
    ↓
用户B打开聊天窗口 → 标记消息已读 → 推送已读回执给用户A
```

## 六、关键技术亮点

1. **实时通信**：基于 WebSocket + STOMP 实现私聊、已读回执、在线状态

5. **密码安全**：BCrypt 加密存储，防止明文泄露
6. **权限控制**：前端路由守卫 + 后端角色校验

## 七、代码优化建议

1. **JWT 认证**：当前使用 mock token，建议替换为真实的 JWT 实现
2. **消息队列**：高并发场景下可引入 Kafka/RabbitMQ 解耦消息推送
3. **缓存优化**：热门场馆信息可使用 Redis 缓存
4. **日志完善**：增加更多业务日志，便于问题排查
5. **API 限流**：防止恶意请求攻击
