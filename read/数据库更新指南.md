# 数据库更新指南：添加置顶和精华功能

## 更新内容
为 `posts` 表添加两个新字段：
- `is_pinned`：是否置顶（BOOLEAN，默认 FALSE）
- `is_featured`：是否精华（BOOLEAN，默认 FALSE）

## 更新方法

### 方法一：使用 MySQL 命令行（推荐）

1. **连接到 MySQL 数据库**
```bash
mysql -u root -p
```

2. **选择数据库**
```sql
USE gymdb;
```

3. **执行更新脚本**
```sql
-- 添加 is_pinned 字段
ALTER TABLE posts 
ADD COLUMN is_pinned BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否置顶' AFTER comment_count;

-- 添加 is_featured 字段
ALTER TABLE posts 
ADD COLUMN is_featured BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否精华' AFTER is_pinned;

-- 添加索引
CREATE INDEX idx_is_pinned ON posts(is_pinned);
CREATE INDEX idx_is_featured ON posts(is_featured);
```

4. **验证更新**
```sql
-- 查看字段
SHOW COLUMNS FROM posts LIKE 'is_pinned';
SHOW COLUMNS FROM posts LIKE 'is_featured';

-- 查看索引
SHOW INDEX FROM posts WHERE Key_name IN ('idx_is_pinned', 'idx_is_featured');
```

### 方法二：使用 MySQL Workbench 或其他 GUI 工具

1. 打开 MySQL Workbench
2. 连接到数据库服务器
3. 选择 `gymdb` 数据库
4. 打开 SQL 编辑器
5. 复制并执行以下 SQL：

```sql
ALTER TABLE posts 
ADD COLUMN is_pinned BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否置顶' AFTER comment_count;

ALTER TABLE posts 
ADD COLUMN is_featured BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否精华' AFTER is_pinned;

CREATE INDEX idx_is_pinned ON posts(is_pinned);
CREATE INDEX idx_is_featured ON posts(is_featured);
```

### 方法三：使用 SQL 文件执行

1. **使用命令行执行 SQL 文件**
```bash
mysql -u root -p gymdb < gymbackend/src/main/resources/database_update_manual.sql
```

2. **或者在 MySQL 命令行中执行**
```sql
source gymbackend/src/main/resources/database_update_manual.sql
```

### 方法四：Hibernate 自动更新（不推荐用于生产环境）

如果你的 `application.properties` 中配置了：
```properties
spring.jpa.hibernate.ddl-auto=update
```

Hibernate 会在应用启动时自动检测实体变化并更新数据库结构。但是：
- ⚠️ **不会自动添加索引**
- ⚠️ **生产环境不推荐使用**
- ✅ **开发环境可以使用**

如果使用此方法，启动应用后需要手动添加索引：
```sql
CREATE INDEX idx_is_pinned ON posts(is_pinned);
CREATE INDEX idx_is_featured ON posts(is_featured);
```

## 注意事项

1. **备份数据**：在执行更新前，建议备份数据库
   ```bash
   mysqldump -u root -p gymdb > gymdb_backup_$(date +%Y%m%d).sql
   ```

2. **字段已存在**：如果字段已存在，执行 ALTER TABLE 会报错，可以忽略

3. **索引已存在**：如果索引已存在，执行 CREATE INDEX 会报错，可以忽略

4. **默认值**：所有现有记录的 `is_pinned` 和 `is_featured` 都会自动设置为 `FALSE`

## 验证更新成功

执行以下查询确认更新成功：
```sql
SELECT 
    COLUMN_NAME,
    COLUMN_TYPE,
    IS_NULLABLE,
    COLUMN_DEFAULT,
    COLUMN_COMMENT
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = 'gymdb'
  AND TABLE_NAME = 'posts'
  AND COLUMN_NAME IN ('is_pinned', 'is_featured');
```

应该看到两行结果，分别对应 `is_pinned` 和 `is_featured` 字段。

## 回滚（如果需要）

如果更新后需要回滚，执行：
```sql
-- 删除索引
DROP INDEX idx_is_pinned ON posts;
DROP INDEX idx_is_featured ON posts;

-- 删除字段
ALTER TABLE posts DROP COLUMN is_featured;
ALTER TABLE posts DROP COLUMN is_pinned;
```
