# 动态热门和精华判断说明

## 功能概述

系统提供了三种动态标记方式：
1. **置顶（Pinned）**：管理员手动设置，显示在最前面
2. **精华（Featured）**：管理员手动设置，标记优质内容
3. **热门（Hot）**：系统自动计算，基于互动数据和时间衰减

## 判断逻辑

### 1. 置顶（isPinned）
- **设置方式**：管理员手动设置
- **显示位置**：动态列表最前面
- **API接口**：`PUT /api/posts/{id}/pin?currentUserId={userId}&pinned=true`
- **权限要求**：管理员权限

### 2. 精华（isFeatured）
- **设置方式**：管理员手动设置
- **显示位置**：在精华排序中优先显示
- **API接口**：`PUT /api/posts/{id}/feature?currentUserId={userId}&featured=true`
- **权限要求**：管理员权限

### 3. 热门（isHot）- 自动计算

热门动态由系统根据以下规则自动判断：

#### 热门分数计算公式
```
热门分数 = (点赞数 × 2 + 评论数 × 3) - 时间衰减
时间衰减 = (当前时间 - 创建时间) / 24小时 × 10分
最终分数 = max(0, 基础分数 - 时间衰减)
```

#### 热门判断条件（满足任一条件即为热门）

**条件1：热门分数达到阈值**
- 热门分数 >= 20 分

**条件2：高互动数**
- 点赞数 >= 10 且 评论数 >= 5

**条件3：24小时内高互动**
- 创建时间在24小时内 且 总互动数（点赞+评论）>= 15

### 示例

#### 示例1：新发布的高互动动态
- 点赞数：8
- 评论数：5
- 发布时间：2小时前
- 热门分数：(8 × 2 + 5 × 3) - 0 = 31分
- **结果**：✅ 热门（满足条件1：分数 >= 20）

#### 示例2：高互动数动态
- 点赞数：12
- 评论数：6
- 发布时间：3天前
- 热门分数：(12 × 2 + 6 × 3) - (72小时/24 × 10) = 42 - 30 = 12分
- **结果**：✅ 热门（满足条件2：点赞 >= 10 且 评论 >= 5）

#### 示例3：24小时内高互动
- 点赞数：10
- 评论数：6
- 发布时间：18小时前
- 热门分数：(10 × 2 + 6 × 3) - 0 = 38分
- **结果**：✅ 热门（满足条件3：24小时内且互动数 >= 15）

#### 示例4：低互动动态
- 点赞数：3
- 评论数：2
- 发布时间：1小时前
- 热门分数：(3 × 2 + 2 × 3) - 0 = 12分
- **结果**：❌ 不是热门（所有条件都不满足）

## API 使用

### 获取动态列表（支持排序）

```http
GET /api/posts?currentUserId={userId}&sortBy={sortType}
```

**排序类型（sortBy）**：
- `latest`：最新排序（默认）
- `hot`：热门排序（按热门分数）
- `featured`：精华排序（精华优先）

### 设置置顶（管理员）

```http
PUT /api/posts/{id}/pin?currentUserId={userId}&pinned=true
```

### 设置精华（管理员）

```http
PUT /api/posts/{id}/feature?currentUserId={userId}&featured=true
```

## 前端显示

### 标记显示
- 📌 **置顶**：金色标记，显示在动态卡片顶部
- ⭐ **精华**：橙色标记，显示在动态卡片顶部
- 🔥 **热门**：红色标记，显示在动态卡片顶部

### 卡片样式
- 置顶动态：金色边框和背景渐变
- 精华动态：橙色边框和背景渐变
- 热门动态：红色边框和背景渐变

## 数据字段

### PostResponse 包含字段
```java
{
  "isPinned": boolean,      // 是否置顶
  "isFeatured": boolean,    // 是否精华
  "isHot": boolean,         // 是否热门（自动计算）
  "hotScore": integer,      // 热门分数（用于排序）
  "likes": integer,         // 点赞数
  "comments": integer        // 评论数
}
```

## 注意事项

1. **热门是实时计算的**：每次获取动态列表时都会重新计算热门状态
2. **时间衰减**：动态发布越久，热门分数会逐渐降低
3. **管理员权限**：只有管理员可以设置/取消置顶和精华标记
4. **热门自动更新**：当动态的点赞或评论数变化时，热门状态会自动更新
