# 健身场馆预约管理系统 - 项目启动指南

## 📋 项目概述

这是一个**健身场馆预约管理系统**，采用前后端分离架构，提供完整的场馆预约、社交互动、聊天等功能。

### 技术栈

**前端 (gym-vue)**
- Vue 3.5.25 (Composition API)
- TypeScript 5.9.0
- Vite 7.2.4
- Pinia 3.0.4 (状态管理)
- Vue Router 4.6.3
- Element Plus 2.11.9 (UI组件库)
- Axios 1.13.2
- WebSocket (实时聊天)

**后端 (gymbackend)**
- Spring Boot 3.5.8
- Java 17
- MySQL (数据库)
- Spring Data JPA
- Spring Security (安全认证)
- Spring WebSocket (实时通信)
- Maven (依赖管理)

### 核心功能模块

1. **用户认证系统**
   - 用户注册/登录
   - 个人信息管理
   - 权限管理（普通用户/管理员）

2. **场馆管理系统**
   - 场馆列表展示
   - 场馆详情查看
   - 场馆搜索和筛选
   - 管理员：场馆CRUD操作

3. **预约管理系统**
   - 场馆预约
   - 预约冲突检测
   - 预约记录查询
   - 预约取消

4. **社交互动系统**
   - 发布动态（文字、图片、视频）
   - 点赞和评论
   - 动态列表展示

5. **好友系统**
   - 好友请求发送/接收
   - 好友列表管理
   - 好友搜索

6. **实时聊天系统**
   - WebSocket实时通信
   - 消息记录
   - 未读消息提醒

7. **管理员后台**
   - 数据统计
   - 用户管理
   - 场馆管理
   - 社交内容管理
   - 学校管理

## 🚀 启动步骤

### 前置要求

- **Node.js**: ^20.19.0 或 >=22.12.0
- **Java**: 17
- **Maven**: 3.6+ (或使用项目自带的 mvnw)
- **MySQL**: 5.7+ 或 8.0+

### 第一步：数据库准备

1. 启动 MySQL 服务

2. 创建数据库：
```sql
CREATE DATABASE gymdb DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. （可选）数据库配置检查
   - 默认数据库：`gymdb`
   - 默认用户名：`root`
   - 默认密码：`123456`
   - 默认端口：`3306`

   如需修改，请编辑 `gymbackend/src/main/resources/application.properties` 或使用环境变量：
   - `DB_URL`
   - `DB_USERNAME`
   - `DB_PASSWORD`

### 第二步：启动后端服务

1. 打开终端，进入后端目录：
```bash
cd gymbackend/gymbackend
```

2. **Windows 系统**启动：
```bash
.\mvnw.cmd spring-boot:run
```

4. 等待启动完成，看到类似以下信息表示成功：
```
Started GymbackendApplication in X.XXX seconds
```

5. 后端默认运行在：`http://localhost:8080`

**注意**：首次启动会自动创建数据库表结构并注入演示数据，包括：
- 管理员账号：`1001` / 密码：`123456`

### 第三步：启动前端服务

1. 打开**新的终端**，进入前端目录：
```bash
cd gym-vue
```

2. 安装依赖（如果还没安装）：
```bash
npm install
```

3. 启动开发服务器：
```bash
npm run dev
```

4. 前端默认运行在：`http://localhost:5173`

5. 在浏览器中访问：`http://localhost:5173`

## ⚠️ 常见问题

### 1. 后端启动失败

**问题**：数据库连接失败
- 检查 MySQL 服务是否启动
- 检查数据库账号密码是否正确
- 检查 `application.properties` 配置

**问题**：端口被占用
- 修改 `application.properties` 中的 `server.port`

### 2. 前端启动失败

**问题**：依赖安装失败
- 清除缓存：`npm cache clean --force`
- 删除 `node_modules` 和 `package-lock.json`，重新安装

**问题**：无法连接后端API
- 检查后端服务是否启动
- 检查 `vite.config.ts` 中的代理配置

### 3. 数据库相关问题

**问题**：表结构未创建
- 检查 `application.properties` 中 `spring.jpa.hibernate.ddl-auto=update`
- 检查数据库是否存在
